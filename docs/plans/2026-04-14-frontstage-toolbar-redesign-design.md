# Frontstage 界面重构设计文档

> 日期：2026-04-14  
> 主题：底部工具栏清理 + 侧边栏 Dock 化 + 编辑器右键菜单  
> 方案：A（双管齐下）

---

## 1. 设计目标

- **清理底部输入栏**：移除影响美观的 4 个功能按钮（修订、批注、评论线程、古典评点），让底部回归纯粹的 AI 对话输入框。
- **活用侧边栏**：当前侧边栏除"幕后"按钮外几乎空置，将其改造为垂直工具 Dock，收纳上述 4 个功能。
- **增强情境化操作**：在正文编辑器中实现右键菜单，支持复制/粘贴/全选等基础功能，同时集成 4 个核心写作功能，未来可扩展更多能力。

---

## 2. 侧边栏 Dock 设计

### 2.1 布局

侧边栏从现有的 120px 缩减为 **64px 纯图标 Dock**（展开标签时可选 120px，默认 64px）。

| 位置 | 功能 | 图标 |
|------|------|------|
| 顶部 | 幕后 | `Eye` |
| 中部 | 修订 | `GitBranch` |
| 中部 | 批注 | `StickyNote` |
| 中部 | 评论 | `MessageSquarePlus` |
| 中部 | 评点 | `Quote` |

### 2.2 视觉规范

- 按钮容器：36×36px，`rounded-xl`（8px 半径）
- 默认状态：透明背景，`var(--stone-gray)` 图标
- 悬停状态：`var(--warm-sand)` 背景，`var(--charcoal)` 图标，`active:scale-[0.98]`
- 激活状态：左侧 2px `var(--terracotta)` 竖线（`box-shadow: inset 2px 0 0 0 var(--terracotta)`），背景 `var(--warm-sand)`
- 图标尺寸：20px
- 工具提示：hover 时显示中文标签（`title` 属性或 tooltip）

### 2.3 行为

- 点击"修订"：切换 `isRevisionMode` 状态
- 点击"批注"：切换 `showAnnotationPanel` 状态
- 点击"评论"：切换 `showCommentPanel` 状态
- 点击"评点"：触发 `handleGenerateCommentary`
- 点击"幕后"：调用 `openBackstage()`

## 3. 编辑器右键菜单设计

### 3.1 触发

在 `.ProseMirror` 区域内监听 `contextmenu` 事件，阻止浏览器默认菜单，根据鼠标位置显示自定义菜单。

### 3.2 菜单结构

```
┌─────────────────────────────┐
│  剪切    复制    粘贴       │  ← 基础编辑组
├─────────────────────────────┤
│  ✓ 修订模式                 │  ← 功能组1（开关类）
│  添加批注...                │
│  发起评论...                │
├─────────────────────────────┤
│  生成古典评点               │  ← 功能组2（动作类）
│  全选                       │
└─────────────────────────────┘
```

### 3.3 视觉规范

- 背景：`var(--ivory)`
- 边框：`1px solid var(--warm-sand)`
- 圆角：`rounded-xl`（12px）
- 内边距：6px
- 最大宽度：180px
- 菜单项：`px-3 py-2 rounded-lg text-sm`
- 悬停项背景：`var(--warm-sand)`
- 禁用项：opacity 0.4，cursor not-allowed
- 微交互：`active:scale-[0.98]`

### 3.4 行为细节

- **复制/粘贴/剪切/全选**：调用 document `execCommand` 或 TipTap 内置命令
- **修订模式**：显示 `Check` 图标标记当前状态，点击切换
- **添加批注 / 发起评论**：
  - 有文本选区：直接打开对应的面板（`showAnnotationPanel=true` 或 `showCommentPanel=true`），并将当前选区信息同步到批注/评论状态
  - 无文本选区：显示禁用态，hover 或点击时 toast 提示"请先选中文本"
- **生成古典评点**：
  - 触发 `handleGenerateCommentary`
  - 生成中菜单项显示 `Loader2` spinner 并 disabled
- **关闭方式**：点击菜单项后自动关闭；点击页面其他区域关闭；按 Escape 关闭

### 3.5 位置逻辑

- 菜单默认在鼠标右键位置弹出
- 边界检测：如果靠近窗口右/下边缘，向左/上偏移，确保菜单不超出视口

## 4. 底部输入栏清理

### 4.1 清理前

`chat-input-right` 包含：修订、批注、评论、评点、发送，共 5 个按钮。

### 4.2 清理后

`chat-input-right` 仅保留：**发送按钮**。

底部栏结构：
- 左侧：`chat-toggle-btn`（展开历史）+ `model-status-dot`（模型状态）
- 中间：`chat-textarea`（输入框）
- 右侧：`chat-send-btn`（发送按钮）

批注面板和评论面板的展开/收起行为保持不变，由侧边栏 Dock 和右键菜单控制。

---

## 7. 设计决策

- **侧边栏宽度 64px**：Dock 风格更现代、占用更少水平空间，最大化正文区域
- **右键菜单分组**：基础编辑与高级功能用分隔线区分，降低认知负担
- **底部纯输入**：让 AI 对话栏回归单一职责，界面更简洁
